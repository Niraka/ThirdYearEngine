Detailed Description

The configuration manager is responsible for loading the engine configuration from the "engine_properties.xml" file.

This manager has the following functionality:

  • Properties may be accessed from anywhere
  • Properties can be loaded and reloaded at any time with the loadProperties function
  • The name of the properties file is fixed
  • Classes may subscribe to the configuration manager to receive notification of properties changing

IMPORTANT: The configuration manager only represents the DESIRED configuration, not the active configuration. Of course, once the configuration has been loaded and distributed, settings may be changed by the program or the user to best accommodate the runtime environment. Take the window style for example. The configuration may launch in windowed, but the user may switch to fullscreen.
